# 🪄 とんがり帽子風 魔法構築ドローウェブアプリ（プロトタイプ）

『とんがり帽子のアトリエ』の世界観にインスパイアされた、スマホのブラウザで魔法陣を描いて遊ぶ非公式のデジタル玩具（プロトタイプ）です。
Godot Engineで制作し、HTML5（WebAssembly）として書き出すことで、インストール不要のWebリンクだけで手軽に遊べる環境を構築します。

クエストやクリア目標はなく、**「自分でパーツを組み合わせて魔法をクラフトする手触り」**と**「それに応じた世界のリアクション」**を楽しむサンドボックス的な仕様です。

---

## 📱 アプリケーション概要

- **ターゲットプラットフォーム:** スマートフォン（iOS / Android）のブラウザ（Safari, Chrome等）
- **配信形式:** HTML5 Webアプリ（GitHub Pages等でホスティング、ホーム画面追加に対応）
- **画面向き:** 縦画面固定
- **開発環境:** Godot Engine 4.x (GDScript)

---

## 🎮 コアゲームプレイループ（魔陣構築プロセス）

本作の魔法は、原作通り「紋（中央）」「矢（周り）」「陣（外枠の円）」の3つの要素が揃うことで初めて発動します。プレイヤーは指を離しながら、一筆書きを重ねて魔陣を「構築」していきます。

[ 1. 中央に「紋」を描く ] ─── 属性（炎・水・光・風）が決定
│
▼
[ 2. 周囲に「矢」を描く ] ─── 形状・大きさ（威力）・方向が決定
│
▼
[ 3. 全体を「陣（円）」で囲う ]
│
▼
[ 4. 円が閉じられた瞬間にシステムが自動判定 ]
┌─────────┴─────────┐
▼ (成功)                ▼ (失敗: 円が歪んでいる/要素が足りない)
[ 属性魔法が動的に発動！ ] [ 魔陣がパチパチと明滅して消滅 ]
[ ふでむしが大喜び ]     [ ふでむしが首を傾げる / 暴発に備えて身構える ]

## 🛠️ 機能要件

### 1. メイン画面（キャンバス）
- **描画エリア:** 画面の大部分を占める無地のキャンバス。
- **ふでむし配置エリア:** 画面の隅に常駐するふでむしの2Dスプライト。プレイヤーが描いている線の動きをリアルタイムにじっと目で追う。

### 2. マルチ・ストローク・ドローシステム（`Line2D`）
- 画面をなぞると指の軌跡に合わせてインク風の線が描画される。
- 指を離しても線は消えず、キャンバスに残り続ける（魔円手帳に書き溜めている状態）。

### 3. 動的パラメータ解析 ＆ 魔陣認識システム
最後のストローク（一筆）が「円（陣）」であると認識された瞬間に、キャンバス上のすべての線を以下の手順で一括評価する。

1. **【陣（外枠）の評価】**
   - 最後に描かれた円の「真円度（中心からの半径のばらつき）」を計算。
   - 歪みが少なければ「安定度100%」、歪んでいると魔法の威力減衰や暴発（煙を吹いて消える）確率が上がる。
2. **【紋（中央）の評価】**
   - 円の内側の中心付近にある図形を認識し、属性（○=光、△=火、□=水、その他=風など）を決定。
3. **【矢（周囲）の評価】**
   - 紋の周りに描かれた「直線（矢）」を検出。
   - 矢の**「長さ（始点から終点の距離）」**を魔法の大きさ（威力・スケール）に変換。
   - 矢の**「向き（角度ベクトル）」**を、魔法エフェクトが放たれる物理的な方向にそのまま連動させる。

## 🪄 インタラクション仕様（属性とリアクション）

| 要素 | 役割 | ゲーム内での処理・パラメータ連動 |
| :--- | :--- | :--- |
| **紋（中央）** | **属性の決定** | 描いた図形に応じて、発動するエフェクト（光・火・水・風）が切り替わる。 |
| **矢（周囲）** | **強さと方向** | 線の**「長さ」**でエフェクトのサイズ・射程が変化。<br>線の**「向き（角度）」**でエフェクトが飛んでいく物理方向が決定。 |
| **陣（外枠）** | **魔法の閉鎖・発動** | 円が閉じられた瞬間に自動評価。円の**「正確さ」**でエフェクトの輝きや維持時間が変化。 |

### ふでむしのリアクション（`AnimationPlayer`）
- **描いている最中:** ソワソワしながらペンの動きを追う。
- **「陣」が閉じられた時:** - 成功時は魔法の属性に合わせて大喜びする（火なら熱がる、水なら水しぶきで飛び跳ねる）。
  - 失敗時はあくびをしたり、首を傾げたり、暴発を恐れて縮こまったりする。

## 🚀 開発ロードマップ

- [ ] **Phase 1: 複数線の描画と「円（陣）」の検知トリガー**
  - Godotで画面をなぞって複数の `Line2D` を描き溜められるようにする。
  - 「最後に描いた線が円だったら、発動トリガーを引く」という判定の土台を実装。
- [ ] **Phase 2: 線の物理パラメータ（長さ・向き・正確さ）抽出**
  - 描いた直線の「長さ」と「角度」を計算してベクトル化するロジックの実装。
  - 円の「中心座標」を割り出し、半径のばらつきから正確さを％で出す計算ロジックの実装。
- [ ] **Phase 3: 属性エフェクトの動的生成とWeb書き出しテスト**
  - 判定結果（方向・大きさ）に応じて、パーティクル（`GPUParticles2D`）を動的に変化させて放出。
  - GodotからHTML5へのエクスポートをテストし、スマホのブラウザで実際に線が引けるか手触りを検証。
