> 最終更新: 2026年4月14日

## モデルファミリー概要

Claudeは Anthropic が開発した大規模言語モデルファミリー。現行のClaude 4世代では、用途に応じた3つのモデルラインが提供されている。すべてのモデルがテキスト・画像入力、テキスト出力、多言語対応、ビジョンをサポート。

## 各モデルの特徴と使い分け

### Claude Opus 4.6

公式の説明: *「エージェントとコーディングのための、最もインテリジェントな広く利用可能なモデル」*

- **強み**: 最高水準の推論能力、複雑なコーディングタスク、エージェント型ワークフロー
- **コンテキスト**: 1Mトークン（約75万語相当）の超長文入力に対応
- **出力**: 128kトークンの長い出力に対応（バッチAPIでは最大300k）
- **用途**: 複雑なリファクタリング、大規模コードベースの分析、高度な推論タスク
- **GitHub Copilotでの倍率**: 3x（1リクエスト = プレミアムリクエスト3回分消費）

### Claude Sonnet 4.6

公式の説明: *「速度とインテリジェンスの最高の組み合わせ」*

- **強み**: Opusに迫る知能で高速応答、日常的な開発タスクに最適
- **コンテキスト**: 1Mトークン対応
- **出力**: 64kトークン（バッチAPIでは最大300k）
- **用途**: 日常的なコーディング支援、コードレビュー、ドキュメント生成、一般的な質問応答
- **GitHub Copilotでの倍率**: 1x（標準消費）

### Claude Haiku 4.5

公式の説明: *「フロンティアに近いインテリジェンスを持つ最速モデル」*

- **強み**: 最も高速で低コスト、シンプルなタスクに最適
- **コンテキスト**: 200kトークン（他の2モデルの1/5）
- **出力**: 64kトークン
- **用途**: 簡単なコード補完、定型的な質問、大量処理（バッチ）、コスト重視の用途
- **GitHub Copilotでの倍率**: 0.33x（3リクエストで1回分の消費）

## Extended Thinking と Adaptive Thinking

### Extended Thinking
3モデルすべてが対応。通常の応答に加えて、モデルが内部で段階的な推論プロセスを実行してから回答する機能。数学、論理パズル、複雑な分析など、深い推論が必要なタスクで精度が向上する。

### Adaptive Thinking
Opus 4.6 と Sonnet 4.6 のみ対応。タスクの複雑さに応じて推論の深さを自動的に調整する機能。簡単な質問には即座に回答し、複雑な問題には時間をかけて推論する。

## 使い分けの目安

| ユースケース | 推奨モデル | 理由 |
|-------------|-----------|------|
| 大規模リファクタリング | Opus 4.6 | 最高の推論能力、128k出力 |
| エージェント型の自律タスク | Opus 4.6 | 最もインテリジェント |
| 日常のコーディング支援 | Sonnet 4.6 | 速度と知能のバランス |
| コードレビュー・PR対応 | Sonnet 4.6 | 高速で十分な精度 |
| 簡単な質問・補完 | Haiku 4.5 | 最速・低コスト |
| 大量バッチ処理 | Haiku 4.5 | コスト効率が最も高い |
