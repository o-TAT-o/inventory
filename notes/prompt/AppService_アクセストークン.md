Azure AppService For Container のEasy Authを使ったVueアプリでセキュアにアクセストークンを取得して、バックエンドAPIを呼び出すときにヘッダーに付与する方法を検証したいです。

検証用に ./lab/AppService_AcceccToken のディレクトリにVueアプリを作成してください。

AppServiceのEasy Authでは /.auth/me にアクセスすることでストアストレージから認証情報を取得できます。
バックエンドAPIはAPI Managementで保護されており、API ManagementでJWT検証を行う想定です。
この時アクセストークンの扱いでできるだけセキュアでかつパフォーマンスの考慮やベストプラクティスを考えて検証したいです。

## 案

1. /.auth/me から取得したアクセストークンをlocalstorageにキャッシュしてAPIコール時にヘッダーに付与
2. APIをコールするたびに /.auth/me にアクセスしてアクセストークン取得 → APIヘッダーに付与
3. /.auth/me から取得したアクセストークンをrefで状態管理している変数に保持してAPIコール時にヘッダーに付与
4. Session Storageでアクセストークンを管理
5. HttpOnly Cookieでアクセストークンを管理

その他

## 制限
- Piniaなどの状態管理ライブラリは使用しません
- APIコールはAxiosを使用します。
- VueはVue3を使用します
- アクセストークンの取得やAPIコールを行うためのAxiosインスタンスの作成はVueライブラリを作成し、そこで対応します。実際の画面を作成するVueアプリではこのライブラリを使ってAPIコールを行います。つまりVueアプリ側ではアクセストークンの扱い方を知らなくてもアクセストークンの取得やリフレッシュなどが勝手に行われます。
